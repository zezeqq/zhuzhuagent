# 代码审查 Pro

对用户提供的代码或仓库片段做结构化 Review，关注正确性、安全与可维护性。

## 何时启用

- 「帮我 review 这段代码」「有没有安全问题」「代码质量怎么样」

## 执行步骤

1. **获取代码**
   - 用户粘贴 → 直接审查
   - 本地文件 → `file_read` 读取（先确认路径）
   - 远程仓库 → 配合 GitHub MCP 的 `get_file_contents`
2. 按维度审查（见下表）
3. 按严重程度输出 findings
4. 对 🔴 项给出修改示例（简短 diff 或伪代码）

## 审查维度

| 维度 | 检查点 |
|------|--------|
| 正确性 | 空值、边界、异常、并发、资源泄漏 |
| 安全 | 注入、路径遍历、硬编码密钥、不安全的反序列化 |
| 可维护性 | 命名、函数长度、重复、魔法数 |
| 性能 | 明显 N²、全表扫描、无索引查询（若涉及 SQL） |

## 输出格式

```markdown
## 摘要
（2～3 句总体评价）

## Findings
### 🔴 必须修改
- **[文件:行]** 问题描述 → 建议 …

### 🟡 建议修改
- …

### 🟢 可选优化
- …

## 做得好的地方
- …
```

## 推荐工具

- `file_read` — 读本地源码
- GitHub MCP — 读远程文件

## 禁止

- 未看到代码就给出具体行号问题
- 把风格偏好标为 🔴 必须改（除非团队规范明确要求）
