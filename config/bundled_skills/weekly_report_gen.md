# 周报 / 月报

根据用户提供的工作事实，生成结构清晰、可汇报的 Markdown；可选导出 Word。

## 何时启用

- 写周报、月报、季度总结、向领导汇报进展

## 执行步骤

1. **收集信息**（缺则追问，不要编造）：
   - 汇报周期（本周/本月）
   - 完成事项（可量化）
   - 进行中任务与进度
   - 风险/需协调问题
   - 下期计划
2. 按下方模板生成 **Markdown**
3. 若用户要 Word：调用 `office_word_create`，标题为「XX周报-YYYYMMDD」
4. 若用户要存档：调用 `file_write` 写入 `exports/` 下 `.md` 文件

## 推荐内置工具

- `office_word_create` — 生成 docx
- `file_write` — 保存 Markdown

## 输出模板

```markdown
# 工作汇报（YYYY-MM-DD ~ YYYY-MM-DD）

## 一、本期完成
1. 【事项】成果与数据…

## 二、进行中
| 任务 | 进度 | 预计完成 | 备注 |

## 三、风险与问题
- …

## 四、下期计划
1. …

## 五、需协调/support
- …
```

## 写作要求

- 动词开头、尽量量化（百分比、数量、日期）
- 区分「已完成」与「进行中」
- 无数据时写「待用户补充」，不虚构 KPI
