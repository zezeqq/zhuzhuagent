# 深度研究 Pro

对主题进行多轮检索、交叉验证，输出带 **来源链接** 的结构化研究报告。

## 何时启用

- 行业调研、技术选型、竞品对比、政策解读、文献综述

## 前置

- 优先使用 **Fetch MCP**（`mcp__fetch__*`）抓取公开网页
- 若有 **Brave Search / 搜索类 MCP**，用于发现 URL
- 工具不可用时：明确告知用户，仅基于已有知识做 **标注「未联网验证」** 的参考回答

## 执行流程

1. **澄清范围**：主题、深度、时间范围、语言、是否要对比表
2. **拆分子问题** 3～5 个可检索问句
3. **逐条检索**：每个子问题至少 1 次工具调用，记录 URL 与抓取时间
4. **交叉验证**：矛盾信息并列出处，说明更可信一方及理由
5. **成文**：摘要 → 要点 → 结论 → 参考链接

## 输出结构

```markdown
# 研究报告：[主题]

## 摘要（200 字内）

## 核心发现
1. … [来源1](url)

## 详细分析
### 子问题 A
…

## 结论与建议

## 参考来源
| # | 标题 | URL | 访问日期 |
|---|------|-----|----------|

## 局限性
- 未覆盖 … / 部分来源无法访问
```

## 推荐 MCP

- `fetch` — 抓取网页正文
- `brave-search` 或同类 — 发现链接（以 MCP 中心实际 ID 为准）

## 禁止

- 伪造 URL 或引用未实际访问的页面
- 把推断写成已证实事实（须标「推测」）
