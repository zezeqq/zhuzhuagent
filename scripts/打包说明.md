# Buddy 打包与分发说明

## 一、在本机打包

```powershell
cd D:\Lifesoftware\pythonandpycharmdata\DNA\PythonProject

# 建议使用项目虚拟环境
.venv\Scripts\python.exe -m pip install -r requirements.txt

# 打包（会自动检查 mcp / PyInstaller）
powershell -ExecutionPolicy Bypass -File scripts\build.ps1
```

输出目录：`dist\🐷🐷Buddy\`（名称以 `core/app_identity.py` 为准）

---

## 二、发给其他电脑时拷什么

| 要拷 | 不要只拷 |
|------|----------|
| **整个** `dist\🐷🐷Buddy\` 文件夹 | 单独一个 `.exe` |

PyInstaller 使用 **onedir** 模式，exe 依赖同目录下 `_internal` 等文件。

**不要**把开发机上的 `data\database.sqlite` 直接覆盖给用户（含你的 API Key 和私人对话）。  
可只拷贝空的 `data\exports` 等子目录结构。

## 三、目标电脑最低要求

| 项目 | 是否必须 | 说明 |
|------|----------|------|
| Windows 10/11 64 位 | 是 | |
| Buddy 整个文件夹 | 是 | 解压/拷贝即可运行，**无需安装 Python** |
| AI 模型 API Key | 是（要用 AI） | 设置 → 模型，每台机器单独配置 |
| **Node.js LTS** | MCP 必须 | 提供 `npx`，用于启动 Filesystem 等 MCP Server |
| GitHub / Brave 等 Token | 可选 | 在 MCP 配置里填写 |
| uv / uvx | 可选 | 仅 Web Fetch 预设默认需要 |

## 四、目标电脑首次使用（推荐顺序）

1. 安装 [Node.js LTS](https://nodejs.org)（若要用 MCP）
2. 双击运行 `🐷🐷Buddy.exe`
3. **设置 → 模型**：填写 DeepSeek 等 API Key 并启用
4. **设置 → 安全中心**：开启「MCP 外部工具」；网络类 MCP 还需「网络访问」
5. 点击聊天底栏 **MCP** → **Test & reload**
   - 成功示例：`Connected 1 server(s), 0 failed, 14 tool(s) available.`
   - 底部状态栏应显示：`MCP：14 工具`
6. 模式选 **Craft**，发消息测试

### MCP 快速测试话术

```
请用 mcp__filesystem__list_directory 列出：
D:\...\🐷🐷Buddy\data\exports
（路径改为本机 exports 实际路径）
禁止用 file_list/file_read。
```

变更 Tab 里应出现 `mcp__filesystem__*` 工具步骤。

## 五、打包已内置 vs 仍需用户安装

| 能力 | 打包进 exe |
|------|------------|
| Buddy 主程序、内置工具、Skill 提示词 | 是 |
| MCP **Python 客户端**（`mcp` 库） | 是（build.spec 已包含） |
| MCP **Server 进程**（Filesystem 等） | **否**，运行时通过 `npx` 下载/启动 |
| Node.js | **否**，用户本机安装 |
| 各厂商 API Key | **否**，用户自行配置 |

## 六、常见问题

### 1. 启动报 `MCP SDK not installed`

- **开发环境**：在项目 venv 执行 `pip install mcp` 或 `pip install -r requirements.txt`
- **打包版**：用最新 `scripts/build.ps1` 重新打包（旧包可能未打入 mcp）

### 2. MCP 显示「未连接 / 0 工具」

- 本机是否安装 Node.js？命令行执行 `npx --version` 应有输出
- MCP 对话框 → Test & reload，查看 `✗ filesystem: ...` 后的错误
- Filesystem 允许目录默认是 exe 旁的 `data\exports`

### 3. 模型说没有 `mcp__` 工具

- 确认状态栏 **MCP：N 工具**（N > 0）
- 模式必须是 **Craft**（Ask 不调工具）
- 重启应用后再试

### 4. Web Fetch 连不上

- 预设使用 `uvx`，需安装 [uv](https://github.com/astral-sh/uv)，或在 MCP 高级 JSON 中改为 npx 版 server
